## 目标
- 前端提供完整的登入/登出体验，令牌安全与自动刷新，登录态门控。
- 后端提供稳定的认证接口与登出撤销机制。
- 学习记录云端化：造句成功时上报；复盘页从后端读取；首次登录同步本地记录。

## 后端实现
### 认证与登出
- 接口：
  - `POST /api/auth/login` 返回 `accessToken` 与 `refreshToken`
  - `POST /api/auth/refresh` 使用 `refreshToken` 换取新令牌
  - `POST /api/auth/logout` 撤销当前设备的刷新令牌；可扩展“全部设备登出”
- 安全：
  - Access Token（短期，含 JTI）+ Refresh Token（持久，轮换）
  - 撤销策略：`RefreshToken` 入库，登出时删除当前设备记录；后续引入黑名单/JTI检查
- 依赖：`PrismaService` + `argon2` + `jsonwebtoken`

### 学习记录
- 数据模型：`WordMastery(userId, word, definition, partOfSpeech, example, userSentence, masteredAt)`（已在 `api/prisma/schema.prisma`）
- 接口：
  - `POST /api/learning/mastery`（需鉴权）造句正确后写入一条记录
  - `POST /api/learning/mastery/list`（需鉴权）按时间倒序返回当前用户记录
- 守卫：所有学习接口使用 `JwtGuard` 读取 `req.user.id`

## 前端实现
### 登入/登出
- 认证页组件：`components/Auth.tsx`（邮箱+密码登录/注册），成功后保存 `linguaCraft_token`
- 根组件门控：未登录显示认证页；登录后显示 `Onboarding`/`Learning`/`Review` `App.tsx`
- 登出：
  - Header 增加“登出”按钮，调用 `POST /api/auth/logout`，清除本地 token，返回登录页
  - 现有“返回首页”逻辑同步清理 token（已在 `App.tsx`）
- 代理：前端 `/api` 通过 Vite 代理到 Nest API `vite.config.ts`

### 学习记录联动
- 上报：在造句正确回调中调用 `POST /api/learning/mastery`（已在 `App.tsx` 整合）
- 复盘：`ReviewList` 首次打开从后端拉取 `mastery/list` 映射展示（已接入）
- 首次登录同步：检测本地 `linguaCraft_mastered`，批量调用 `mastery` 接口上报；成功后本地与云端保持一致

### 令牌与拦截器
- 统一请求拦截器：附带 `Authorization: Bearer <token>`；遇 401 自动调用 `/api/auth/refresh`，刷新失败则登出
- 错误提示与重试：登录失败与学习接口错误友好提示；对 5xx 提供重试
