## 现状确认
- 路由守卫仍在：`frontend/router/AuthGuard.tsx:5-9` 对 `/`、`/learn/:level`、`/review` 生效（见 `frontend/App.tsx:33-41`），`token` 为空时重定向到 `/login`。
- 401 的强制登出也仍在：各 API 遇到 401 时调用 `forceLogout()`（如 `frontend/services/geminiService.ts:30,56,74,83,92,101`），`AppContext` 监听 `force-logout` 事件并清空 `token`（`frontend/contexts/AppContext.tsx:65-75`）。
- 可能缺失的场景：当当前路由未被 `AuthGuard` 包裹或由于某些渲染时机问题，`token` 变为 `null` 后没有立即跳到登录页。

## 调整方案
- 在顶层 Shell 添加兜底重定向：当 `token` 变为 `null`，无论当前路由是否被 `AuthGuard` 包裹，均 `navigate('/login', { replace: true, state: { from: location } })`。
- 不改变既有 `AuthGuard` 行为，仅增加一个全局 effect 作为兜底，让“在登录态丢失后，再进行任何操作都退回登录页”的特性更稳健。

## 具体改动
- 更新 `frontend/App.tsx` 中的 Shell：
  - 读取 `token`、`useLocation()` 与 `useNavigate()`；
  - 新增 `useEffect` 监听 `token` 变化；`!token && location.pathname !== '/login'` 时执行重定向，并保留来源路径到 `state.from`。
