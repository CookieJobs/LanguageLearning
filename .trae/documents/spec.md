# 答题页面布局修改规范 (Spec)

## 1. 项目背景
当前的答题页面在桌面端采用了“左侧显示单词/问题，右侧显示选项/输入框”的左右分栏布局。用户希望将其统一修改为“上下结构”，使单词/问题展示在上方，答题区域在下方。

## 2. 目标
* 移除所有强制桌面端采用左右布局的 CSS 类。
* 确保顶部区域能够根据内容自适应高度，并带有合适的内边距。
* 确保底部答题区域能够占满剩余垂直空间，并且在内容过长时可以内部滚动。

## 3. 技术方案
涉及修改的核心文件为 `frontend/components/LearningSession.tsx`：
1. **主容器**：移除 `md:flex-row`，仅使用 `flex-col` 保持所有断点下的垂直布局。
2. **顶部区域（Word/Question Display）**：移除 `md:w-[38%]` 和 `md:h-full`，使用 `py-8 md:py-10` 提供内边距。将原本用于分隔左右栏的右边框 `border-r-4` 改为底边框 `border-b-4`，添加 `shrink-0` 确保其不被挤压。
3. **底部区域（Workspace）**：移除 `h-full` 换成 `overflow-hidden`，保证 `flex-1` 分配的剩余空间内能实现完美的子元素滚动（`overflow-y-auto`）。

## 4. UI/UX 影响
修改后，无论在移动端还是桌面端，用户都将获得一致的上下滚动式答题体验。顶部单词区域横向扩展，居中对齐，留出更多水平空间，并为底部答题区留出足够的横向空间以显示较长的句子或选项。
