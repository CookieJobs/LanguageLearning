## 目标
- 前端完整接入登录态体验（刷新令牌、登出、资料编辑、错误处理）。
- 后端增强安全与稳定性（限流、审计、邮箱验证/重置密码），为生产切换到 Postgres/Redis 做好准备。
- 学习数据云端化（掌握进度同步、复盘读取），端到端稳定运行。

## 前端改造
- 登录态拦截器：统一附加 `Authorization`，遇 401 自动调用 `/api/auth/refresh`，失败则清除令牌并跳转登录。
- 登出入口：在 Header 添加登出按钮，调用 `/api/auth/logout` 并清理本地 token。
- 资料编辑页：新增简单的“我的资料”页面，调用 `GET/PATCH /api/me`。
- 错误提示与重试：Auth 与 Learning 请求的失败提示与重试按钮；对 5xx 做友好提示。

## 后端增强
- 限流与审计：对 `/auth/*` 与 `/learning/*` 接口添加基于 Redis 的限流，记录登录失败、刷新、登出等审计日志。
- 邮箱验证码与密码重置：新增 `POST /auth/send-code`、`POST /auth/reset`；接入邮件服务（Resend/SendGrid）。
- Token 安全：刷新令牌轮换策略完善，记录设备信息与 IP 近端校验；Access Token 加 JTI 与黑名单支持。

## 学习数据云端化
- 数据模型：新增 `WordMastery(userId, word, partOfSpeech, definition, userSentence, masteredAt)`。
- 同步策略：前端首次登录批量上传 `localStorage` 数据，后续每次掌握事件实时上报；复盘列表由后端查询。
- 接口：`GET /learning/mastery`、`POST /learning/mastery`（受鉴权）。

## 生产准备
- 数据库切换：将 Prisma datasource 改为 Postgres，迁移脚本到 `migrate deploy`；启用 Redis 限流与会话相关功能。
