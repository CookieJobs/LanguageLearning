## 目标
- 当用户登录态（token）缺失或失效时，点击任意功能后立刻回到登录界面（显示 `Auth`）。

## 总体思路
- 后端继续用 `JwtGuard` 返回 401（现有已满足）。
- 前端增加全局未授权处理：任何接口返回 401 → 触发强制登出事件 → App 清理本地状态并切到登录。
- 同时在交互入口做轻量前置校验：若本地无 token，直接跳转登录。

## 前端改动
- 新增全局事件与工具
  - 文件：`services/authGuard.ts`（新增）
  - 导出 `forceLogout()`：
    - 清理 `linguaCraft_token/linguaCraft_refresh`
    - `window.dispatchEvent(new Event('force-logout'))`
- App 监听强制登出事件
  - 文件：`App.tsx`
  - 在 `useEffect` 中监听 `force-logout`，回调：`setToken(null); setScreen('onboarding'); setLevel(null); setWordQueue([]); setCurrentWordIndex(0)`（与 `handleLogout` 一致）
- 接口统一 401 处理
  - 文件：`services/geminiService.ts`
  - 在所有 `fetch` 统一判断：`if (res.status === 401) { forceLogout(); throw new Error('unauthorized') }`
  - 具体位置：
    - `fetchWordsForLevel`（已读写处 `services/geminiService.ts:12-35`）
    - `evaluateSentence`（`services/geminiService.ts:37-45`）
    - `addMastery`（`services/geminiService.ts:47-61`）
    - `fetchMasteryList`（`services/geminiService.ts:63-69`）
    - `logout`/`getMe`：`getMe` 失败时若 401，同样 `forceLogout()`
- 入口前置校验（轻量）
  - 文件：`App.tsx`
  - 在交互入口（如选择学段、学习页操作、打开“已掌握单词”）前加 `if (!token) dispatch force-logout`（尽量复用已有 `token` gating，避免重复逻辑）。

## 后端确认
- 保持 `@UseGuards(JwtGuard)` 的接口返回 401：
  - `api/src/modules/learning/learning.controller.ts:16-74`
  - 现状已满足，无需改动。

## 验证
- 场景 1：手动删除本地 `linguaCraft_token`，点击“选择学段”或“已掌握单词”，应直接跳到登录。
- 场景 2：token 过期，调用任意 API 返回 401 → 自动强制登出并切到登录。
- 场景 3：正常登录后流程不受影响；退出登录同样清理并返回登录页。

## 备注
- 此实现避免在每个组件散布登出逻辑，统一由 `services` 层触发与 `App` 层接收，实现清晰的单向数据流。
- 后续可将 `forceLogout` 改为 React Context 的 `auth.logout()`，进一步提升可测试性与可维护性。
