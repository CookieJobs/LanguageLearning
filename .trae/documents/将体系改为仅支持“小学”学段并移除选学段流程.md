## 目标
- 前端完全移除“选择学段”与相关路由、组件与状态，仅保留“小学”学段学习。
- 后端与数据库仅保留“小学”学段数据；词库检索固定按“小学”过滤；不再依赖用户 `educationLevel`。
- 现有用户登录后直接进入首页与学习流程；注册后同样无需选学段。

## 前端改动
### 路由与守卫
- 移除 `/onboarding` 路由与页面：`frontend/App.tsx` 删除该 Route。
- 移除 `LevelGuard`：`frontend/router/LevelGuard.tsx` 不再使用；仅保留 `AuthGuard`。
- 首页 `/`、学习页 `/learn`、复习页 `/review` 保留；`/learn/:level` 改为 `/learn`（去掉参数）。

### 页面与组件
- 删除或停用 `OnboardingPage.tsx` 与 `components/Onboarding.tsx`，同时移除首页中“更换学段”入口。
- 在 `HomePage.tsx` 统一显示仪表盘与“开始学习”“查看已掌握”，不暴露学段相关 UI。
- 在 `LearnPage.tsx` 直接按照固定“小学”学段加载会话，无需根据 URL 参数切换。

### 上下文与服务
- `frontend/contexts/AppContext.tsx`
  - 移除 `level` 状态与 `handleLevelSelect` 相关逻辑；固定常量 `PRIMARY`；移除 `linguaCraft_lastLevel/levelSince` 的读写。
  - 所有词库拉取统一调用 `fetchWordsForLevel('Primary')`（或者改造为无参函数 `fetchWords()`）。
  - 移除对 `updateMe({ educationLevel })` 的调用；保留 `getMe` 仅用于 email 与统计。
- `frontend/services/geminiService.ts`
  - 将 `fetchWordsForLevel(level, exclude)` 改为固定按“小学”过滤：
    - 方案 A：保留函数签名但忽略传入 `level`，内部写死为 `Primary`。
    - 方案 B：改为 `fetchWords(exclude)` 无需传 level（同时更新所有调用处）。
- `frontend/pages/LoginPage.tsx`
  - 登录后直接 `navigate('/')`；移除基于 `educationLevel` 的分支逻辑与 `/onboarding` 跳转。
- `frontend/types.ts`
  - 可选择保留枚举但只使用 `PRIMARY`；或将枚举简化为常量；尽量减少横向改动以降低风险。

## 后端改动
- `backend/src/modules/user/user.controller.ts`
  - `GET /api/me` 不再返回 `educationLevel`（或始终返回 `'Primary'` 以向后兼容）。
  - `PATCH /api/me` 忽略 `educationLevel` 字段（可保留参数但不生效，或直接移除 DTO 字段）。
- `backend/src/modules/learning/learning.controller.ts`
  - `POST /api/learning/words` 请求体不再需要 `level`；服务端固定按“小学”过滤返回新词。
- `backend/src/modules/learning/vocab.service.ts`
  - 统一从 `VocabWordLevel` 里筛选 `VocabLevel.code = 'Primary'` 的单词；若无关联则直接使用内置过滤策略。
- 其余接口（评估、掌握、统计）不依赖学段，保持不变。

## 数据库与数据清理（SQLite）
- 保留现有表结构，避免大规模迁移；仅清理数据：
  - 删除所有非 `Primary` 的 `VocabLevel`：
    - `DELETE FROM VocabWordLevel WHERE levelId IN (SELECT id FROM VocabLevel WHERE code <> 'Primary');`
    - `DELETE FROM VocabLevel WHERE code <> 'Primary';`
- 如需更彻底：
  - 将 `UserProfile.educationLevel` 字段置空或忽略；不强制删除列（保持兼容）。

## 验证
- 本地运行后端与前端：
  - 注册新用户 → 登录后直接进入 `/`，点击“开始学习”进入 `/learn` 并成功拉取小学词库。
  - 老用户（含 `educationLevel` 为非 Primary 或标签） → 登录直接进入 `/`；学习同样按小学。
  - 复习页 `/review` 正常显示已掌握；不再提供按学段过滤。

## 回滚策略
- 若需要恢复多学段：保留原 `Onboarding` 组件与 `LevelGuard` 文件，当前改动以注释/删除方式进行，便于 git revert。
- 数据恢复：暂不删除表，仅删除数据；后续可通过种子脚本补回其他学段词库。
