# 登录页慢加载优化实施计划

## Summary
- 目标：优化 `http://localhost:3000/login` 首次进入速度，落实已确认的三项优化：路由懒加载、移除运行时 Tailwind CDN、避免 Google Fonts 外链阻塞。
- 成功标准：登录页首屏不再依赖 `cdn.tailwindcss.com` 和 `fonts.googleapis.com`，并且路由入口不再在访问 `/login` 时同步加载 Home/Learn/Review 页面模块。

## Current State Analysis
- 当前 `frontend/index.html` 在 `<head>` 中同步引入 Tailwind CDN 与 Google Fonts，存在外网请求阻塞风险：
  - `frontend/index.html` 第 7 行：`https://cdn.tailwindcss.com`
  - `frontend/index.html` 第 91 行：`https://fonts.googleapis.com/...`
- 当前 `frontend/App.tsx` 采用静态导入页面组件，访问 `/login` 时也会下载其他路由页面模块：
  - 静态引入：`HomePage`、`LearnPage`、`ReviewPage`、`LevelSelectPage`
- 当前项目尚无本地 Tailwind 构建文件（未发现 `tailwind.config.*`、`postcss.config.*`、项目级 CSS 入口文件）。

## Proposed Changes

### 1) 路由级懒加载（减少 `/login` 首包体积）
- 文件：`frontend/App.tsx`
- 变更内容：
  - 将页面级组件改为 `React.lazy` 动态导入（至少包含 `HomePage`、`LearnPage`、`ReviewPage`、`LevelSelectPage`，`LoginPage` 也改为懒加载以统一路由分包策略）。
  - 在 `<Routes>` 外包裹 `React.Suspense`，提供轻量 fallback，确保加载过程可控。
- 目的：访问 `/login` 时只加载登录所需代码，避免同步拉取学习与复习页面模块。

### 2) 从运行时 CDN 切换到本地 Tailwind 构建（去除阻塞外链脚本）
- 文件：`frontend/package.json`、`frontend/package-lock.json`
- 变更内容：
  - 增加 Tailwind 本地构建依赖（`tailwindcss`、`postcss`、`autoprefixer`）。
- 文件：`frontend/tailwind.config.cjs`（新增）
- 变更内容：
  - 把现有 `index.html` 里的 `tailwind.config` 主题扩展迁移到本地配置（颜色、字体、动画、keyframes、阴影等）。
  - 配置 `content` 扫描路径覆盖 `App.tsx`、`components/**/*.tsx`、`pages/**/*.tsx`、`contexts/**/*.tsx`、`router/**/*.tsx` 等。
- 文件：`frontend/postcss.config.cjs`（新增）
- 变更内容：
  - 配置 Tailwind + Autoprefixer 处理链路。
- 文件：`frontend/styles.css`（新增）与 `frontend/index.tsx`
- 变更内容：
  - 在 `styles.css` 中添加 `@tailwind base/components/utilities` 与当前全局样式（玻璃态、滚动条、渐变边框等）。
  - 在 `index.tsx` 引入 `./styles.css`。
- 文件：`frontend/index.html`
- 变更内容：
  - 删除 Tailwind CDN `<script>` 与内联 `tailwind.config` 脚本，保留最小化文档壳。
- 目的：避免运行时依赖外部 CDN，提高首屏稳定性与可预测性。

### 3) 字体策略改为本地可用栈（去除 Google Fonts 外链）
- 文件：`frontend/index.html`、`frontend/styles.css`、`frontend/tailwind.config.cjs`
- 变更内容：
  - 删除 Google Fonts `<link>`。
  - 将全局字体改为系统字体栈（本地可用），并同步 Tailwind `fontFamily.sans`。
- 目的：消除 `fonts.googleapis.com` 网络抖动导致的首屏变慢。

## Assumptions & Decisions
- 决策：按“最小风险可验证”实现，不改业务逻辑与接口，仅优化资源加载与打包策略。
- 决策：采用系统字体栈作为“本地字体方案”，不额外引入字体文件，避免新增静态资源管理复杂度。
- 假设：现有 UI 对 Tailwind 工具类依赖较高，因此不做样式体系重写，仅迁移至本地构建方式。
- 假设：`importmap` 与 `@google/genai` 当前不影响 `/login` 关键路径，此轮不作为必须改动项。

## Verification Steps
- 构建验证：
  - 安装前端新增依赖并执行 `npm --prefix frontend run build`，确认可编译通过。
- 运行验证：
  - 启动前端开发服务，访问 `/login`，确认页面可正常渲染与交互（登录表单可用）。
- 网络验证（关键）：
  - 在浏览器网络面板确认不再出现 `https://cdn.tailwindcss.com` 与 `https://fonts.googleapis.com` 请求。
  - 确认访问 `/login` 时不再同步请求 `HomePage.tsx`、`LearnPage.tsx`、`ReviewPage.tsx`、`LevelSelectPage.tsx`（开发态模块请求应按懒加载触发）。
- 回归验证：
  - 登录成功后可正常跳转首页，受保护路由访问与原行为一致。
