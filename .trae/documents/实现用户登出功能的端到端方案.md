## 目标
- 为当前系统增加稳定的“用户登出”功能：服务端撤销刷新令牌、前端清理令牌并返回登录页；受保护接口在登出后返回 401。

## 后端改造
- 接口：`POST /api/auth/logout`
  - 方案 A（推荐，单设备登出）：请求体携带 `refreshToken`；服务端验证并撤销该刷新令牌（从 `RefreshToken` 表删除）。
  - 方案 B（简化，全设备登出）：使用 `JwtGuard` 读取当前用户 `id`，删除该用户全部刷新令牌记录（当前代码已有 `AuthService.logout(userId)` 能力）。
- 建议落地：
  - 控制器：`api/src/modules/auth/auth.controller.ts` 增加 `@UseGuards(JwtGuard)`，或接收 `refreshToken` 并调用 `AuthService.logout(...)`。
  - 服务：`AuthService.logout(userId)` 保持，后续支持“按刷新令牌撤销”（只删除匹配 token 的记录）。
- 返回：`204 No Content` 或 `{ ok: true }`。

## 前端改造
- 存储：登录成功时保存 `accessToken` 与 `refreshToken`（目前只保存 `accessToken`）。
- 按钮：在 Header 增加“登出”按钮（`App.tsx` 已有返回首页逻辑，可复用），点击时：
  - 调用 `POST /api/auth/logout`（带 `Authorization` 或传 `refreshToken`）
  - 清理本地令牌：`localStorage.removeItem('linguaCraft_token')`（以及 `refreshToken`）
  - 重置状态：`setScreen('onboarding')`，显示登录页
- 守卫与拦截器：
  - 统一 fetch 封装：自动附加 `Authorization`
  - 遇 401：清理令牌并跳转登录（透明处理过期）

## 验证步骤
- 登录 → 调用受保护接口（如 `/api/me`）返回 200
- 点击“登出” → 再调用受保护接口返回 401
- 登录 → 学习 → 复盘列表正常读取；登出后受保护端点全部拒绝
