# 英语墙 (English Wall) 实施计划

## 1. 目标与背景
本项目旨在为前端复习页面 (`/review`) 引入“英语墙”功能，通过 3D 视觉和游戏化反馈（灰色砖块到发光金砖的转化）来提升用户背单词的成就感和沉浸感。

## 2. 技术选型与策略
- **UI 渲染**：利用 CSS 3D 转换或轻量级 WebGL 库（如 Three.js）渲染砖墙。考虑到十万级数据的性能，需结合虚拟列表或视口外的视锥体裁剪/懒加载技术。
- **状态管理**：扩展现有的 `vocabularyStore`，确保状态变动能够即时响应到 3D 视图和 UI 组件。
- **数据可视化**：弹窗内的学习曲线使用 Recharts/ECharts 绘制。
- **动画特效**：使用 CSS 动画或 `framer-motion` 处理点亮过渡，粒子特效可通过 `canvas` 实现。

## 3. 阶段划分
- **阶段一：入口与基础架构搭建**
  完成 `/review` 入口的 UI，搭建 `EnglishWall` 组件的骨架和数据接入点。
- **阶段二：3D 场景与核心交互**
  实现砖块的 3D 堆砌、状态区分材质、悬停信息卡片以及点亮粒子特效。
- **阶段三：详情弹窗与业务闭环**
  完成包含图表的单词详情模态框，实现快捷复习操作并与全局状态同步。
- **阶段四：性能优化与多端适配**
  引入懒加载机制解决大数据量渲染问题，并优化移动端触摸体验。

## 4. 风险与缓解方案
- **性能瓶颈**：十万级 DOM 节点会导致页面卡死。
  *缓解*：强制使用虚拟化/按需渲染，单面墙可视节点限制在 500 以内。
- **3D 交互在移动端的兼容性**：悬停（hover）在移动端不存在。
  *缓解*：在移动端将首次点击作为“查看悬停卡片/选中”，再次点击或长按唤起详情弹窗。
