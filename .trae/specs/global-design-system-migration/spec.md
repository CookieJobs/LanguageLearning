# 全局设计系统迁移规范

## 为什么 (Why)
为了将产品视觉风格统一为新的“活泼与大胆”（类多邻国风格），确保品牌一致性并提升用户参与度。目前混合存在的“AI风格”渐变/玻璃拟态与新的扁平/3D风格导致了割裂的用户体验。

## 变更内容 (What Changes)

### 1. 设计令牌 (Tailwind 配置)
- **颜色**: 将 `duo` 色板（绿、蓝、黄、红、灰）标准化为主要语义颜色。
    - 添加 3D 边框所需的特定阴影色（如 `green-dark`, `blue-dark`）。
- **边框**: 添加标准边框宽度（如 `3px`, `4px`）和圆角（`xl`, `2xl`）以实现厚实、友好的外观。
- **阴影**: 移除柔和的彩色阴影。替换为“实心”偏移或使用底部边框来表现深度。
- **排版**: 在所有标题和按钮中强制使用粗体圆角无衬线字体栈。
- **动效**: 标准化按钮按下的 `active:translate-y` 效果和页面切换的简单 `fade-in` / `slide-up` 效果。

### 2. 组件重构
- **按钮 (Button)**:
    - 移除所有渐变/幽灵变体。
    - 实现带 3D 边框的 `duo-primary`, `duo-secondary`, `duo-danger`, `duo-ghost` 变体。
    - 确保一致的高度和内边距。
- **输入框 / 表单元素**:
    - 样式：灰色背景 (`bg-gray-100`)，圆角 (`rounded-xl`)，聚焦时实心边框 (`border-blue-500`)，无发光效果。
    - 错误状态：红色边框和文字，无柔和红色背景。
- **卡片 / 容器**:
    - 移除 `.glass` 和 `.glass-card` 效果。
    - 样式：白色背景，圆角 (`rounded-2xl` 或 `3xl`)，实心浅灰边框 (`border-gray-200` 或 `border-b-4`)。
- **页眉 (Header)**:
    - 实心白色背景。
    - 3D 风格导航按钮。
    - Logo 集成（已完成，但需验证间距）。
- **模态框 / 遮罩层**:
    - 简单的高对比度白色卡片。
    - 带 3D 悬停效果的“X”关闭按钮。

### 3. 页面重构
- **认证页面 (登录/注册)**: 移除背景斑点。使用干净的居中卡片布局或平铺背景。
- **仪表盘 / 首页**:
    - 课程/关卡的网格布局。
    - 使用“块状”风格的进度条。
    - 带粗体图标的“每日连胜”或统计卡片。
- **关卡选择**:
    - 带 3D 悬停状态的大型可点击卡片。
- **复习页面**:
    - 列表项改为可按压的卡片样式。
- **学习总结**:
    - 庆祝动效（彩带）。
    - 粗体统计计数器。
    - “继续”按钮作为主要行动点。

## 影响范围 (Impact)
- **受影响规范**: 无（这是迁移的主规范）。
- **受影响代码**:
    - `frontend/tailwind.config.js`
    - `frontend/index.css`
    - `frontend/components/**/*.tsx`
    - `frontend/pages/**/*.tsx`

## 新增需求 (ADDED Requirements)
### 需求：设计一致性
系统所有交互元素必须使用定义的 `duo` 色板和 3D 交互模型。
- **场景**: 用户悬停在任意按钮上。
- **结果**: 按钮变化（如变亮）但不发光。
- **场景**: 用户点击任意按钮。
- **结果**: 按钮物理下沉 (`translate-y`) 且底部边框减少。

### 需求：响应式与无障碍
- 触摸目标至少为 44px。
- 文本对比度必须符合 WCAG AA 标准（针对新色板检查）。
- 键盘导航的聚焦环必须清晰（实心蓝/绿）。

## 修改需求 (MODIFIED Requirements)
### 需求：视觉风格
**从**: “AI/科技感”（渐变、模糊、细边框）
**到**: “活泼/学习感”（扁平色、厚边框、圆角）

## 移除需求 (REMOVED Requirements)
### 需求：玻璃拟态
**原因**: 与新的扁平/粗体方向不一致。
**迁移**: 将所有 `backdrop-blur` 和半透明白色背景替换为不透明白色卡片。
