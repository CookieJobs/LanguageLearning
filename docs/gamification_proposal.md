# 语言学习应用游戏化方案提案

## 1. 概述
为了提升用户的学习动力与粘性，本方案旨在将枯燥的单词记忆过程转化为具有趣味性的游戏体验。核心思路围绕“付出即有回报”的正向反馈机制，结合**宠物养成**与**模拟经营**两大方向，让用户的每一次学习行为都能在虚拟世界中产生直观的视觉反馈。

---

## 2. 方案一：灵语守护兽 (LinguaGuardian) —— 深度养成系

### 核心概念
每个用户在注册时会获得一颗“灵语蛋”，通过完成每日学习任务孵化出专属的“守护兽”。守护兽的成长状态直接反映用户的学习进度。

### 详细玩法
1.  **孵化与进化**：
    *   **初识**：完成第一组单词学习，蛋孵化为幼年期宠物。
    *   **成长**：随着用户掌握单词数量（Mastery Level）的增加，宠物会获得经验值 (EXP) 并升级。
    *   **进化**：每当用户完成一个大单元或级别（Level），宠物会发生形态进化（例如：从幼龙 -> 少年龙 -> 元素巨龙）。

2.  **喂养机制（复习驱动）**：
    *   **饥饿度**：宠物每天会消耗能量。
    *   **食物获取**：用户必须通过**复习（Review）**旧单词来获得“能量果实”。如果不复习，宠物会变得无精打采，甚至离家出走（暂停服务，需通过强化复习找回）。
    *   **逻辑关联**：复习 = 喂养。强迫用户关注遗忘曲线。

3.  **金币与装扮**：
    *   **赚取金币**：在测验（Quiz）中答对题目、连续签到、完成每日挑战（Daily Quest）可获得金币。
    *   **消费**：购买宠物的小配饰（帽子、眼镜）、房间装饰、不同风格的背景。

4.  **AI 互动（特色功能）**：
    *   **对话练习**：点击宠物，它会用用户**已学过的单词**说一两句简单的鼓励话语或冷笑话。
    *   **心情反馈**：答题错误率高时，宠物会露出困惑表情；连续答对时，宠物会欢呼跳跃。

### 激励循环
学习新词 -> 宠物升级 -> 复习旧词 -> 宠物保持健康 -> 测验挑战 -> 赚金币买装扮 -> 炫耀/分享

## 2. 方案二：词汇庄园 (VocabFarm) —— 可视化经营系

### 核心概念
将单词学习过程具象化为农场经营。每个单词是一颗种子，记忆的稳固程度对应植物的生长阶段。

### 详细玩法
1.  **播种（学习新词）**：
    *   每学习一个新单词，就在农场空地上播下一颗种子。
    *   不同的词性（名词、动词、形容词）对应不同的植物类型（如：名词是南瓜，动词是向日葵）。

2.  **灌溉与施肥（复习机制）**：
    *   **干旱预警**：根据艾宾浩斯遗忘曲线，当单词快要被遗忘时，植物上方会出现“干旱”图标。
    *   **浇水**：完成该单词的复习任务 = 浇水。植物恢复生机并继续生长。
    *   **施肥**：在测验中连续答对，获得“强力肥料”，加速植物成熟。

3.  **收获与市场**：
    *   **成熟**：当单词达到“完全掌握”（Mastered）状态，植物成熟结出果实。
    *   **收获**：用户点击收获，获得大量经验值和金币。
    *   **图鉴**：收获的果实点亮“作物图鉴”（即词汇书成就系统）。

4.  **扩建与装饰**：
    *   使用金币开垦更多荒地（解锁新词库）。
    *   购买风车、稻草人、围栏等装饰物打造个性化庄园。

### 激励循环
播种新词 -> 每日浇水(复习) -> 植物防枯萎 -> 最终收获 -> 扩建庄园 -> 满足收集癖

## 3. 方案三：时空旅者 (Time Traveler) —— 剧情探索系（补充）

### 核心概念
用户扮演一名时空修复者，单词是修复历史的能量。

### 详细玩法
1.  **地图解锁**：初级单词对应“石器时代”，中级对应“中世纪”，高级对应“未来都市”。
2.  **文物修复**：每个单元是一个残破的历史文物，学习单词就是收集碎片修复文物。
3.  **AI 剧情**：每完成一个章节，AI 生成一段基于该章节单词的微小说，作为剧情奖励。

---

## 4. 技术实现建议 (MVP 阶段)

我们建议先从 **方案一 (灵语守护兽)** 切入，因为其逻辑更集中，易于与现有 User 模型结合。

### 后端 (NestJS + MongoDB)
1.  **User Schema 扩展**：
    *   增加 `pet` 字段：包含 `level` (等级), `exp` (经验), `hunger` (饥饿度), `appearance` (外观配置)。
    *   增加 `currency` (金币数量)。
2.  **新增 API**：
    *   `POST /pet/feed`：消耗金币/道具喂食，增加饥饿度上限。
    *   `POST /pet/interact`：与宠物互动，触发 AI 对话。
3.  **Hook 埋点**：
    *   在 `MasteryService` 更新单词进度时，触发 `PetService.addExp()`。
    *   在 `QuizService` 完成测验时，触发 `WalletService.addCoins()`。

### 前端 (React)
1.  **全局宠物组件**：
    *   在页面右下角或专用 Tab 显示宠物。
    *   使用 Lottie 动画或简单的 SVG 动画实现宠物的呼吸、开心、进食动作。
2.  **商店页面**：
    *   简单的网格布局，展示可购买的物品。
3.  **反馈系统**：
    *   答题正确时，宠物弹出气泡给予肯定。

---

## 5. 总结
这两个方案都不需要推翻现有架构。
*   **灵语守护兽**更适合增强**用户粘性**（情感连接）。
*   **词汇庄园**更适合增强**成就感**（可视化积累）。

建议先选择其中一个方向进行 MVP (最小可行性产品) 开发，例如先做一个简单的“答题赚金币，金币买宠物饲料”的闭环。
