# 宠物交互系统 (Pet System) 优化实现文档

## 1. 优化背景与目标
为了提升用户在语言学习过程中的参与感与正向反馈，我们对现有的“灵语守护兽 (Pet System)”进行了深度的前端交互与视觉优化。
主要目标包括：明确操作的成本与收益、增加冷却机制防止滥用、提升宠物面板的视觉层级，并优化响应式布局。

## 2. 界面布局重构 (HomePage)

### 修改前：
- 宠物组件 `<PetDisplay />` 被简单地放置在英雄区域 (Hero) 和数据统计卡片 (Stats) 之间，采用上下堆叠的单列布局。
- 视觉权重较低，缺乏与周围元素的融合感。

### 修改后：
- 采用了 **响应式 CSS Grid 网格布局**：
  - **大屏幕 (Desktop)**：采用 `lg:grid-cols-3` 布局，左侧占 2/3 宽度展示学习行动区和统计卡片；右侧占 1/3 宽度，专门作为宠物的侧边栏展示区。
  - **小屏幕 (Mobile)**：自动折叠为单列布局，保证内容的可读性。
- 宠物区域从页面中部的阻断者，变成了持续陪伴在右侧的“侧边栏伙伴”，更符合游戏化产品的用户习惯。
- 为数据统计卡片（连续打卡、学习进度）添加了悬停提升 (`hover:-translate-y-1`) 和边框变色动画，提升了整体页面的点击欲望。

## 3. 宠物面板交互优化 (PetDisplay)

### 3.1 Feed (喂食) 功能改进
**改动内容：**
- **按钮重设计**：将原来的简单文本按钮替换为包含图标 (`Utensils`, `Coins`) 的复合型按钮。
- **成本提示**：在按钮内部直接以小字体显示 `-10 Coins`。
- **动态状态**：当金币不足或正在喂食时，按钮置灰且不可点击。
- **悬停预览 (Tooltip)**：引入了 `group-hover` 机制，鼠标悬停时弹出黑色提示框，明确显示：
  - 消耗：`-10 金币` (红色高亮)
  - 收益：`+20 饥饿度` (绿色高亮)
- **视觉反馈**：点击喂食后，宠物头像会触发放大的弹跳动画 (`scale-110 rotate-3`)，并飘出一个苹果 🍎 图标。

### 3.2 Play (互动) 功能改进
**改动内容：**
- **冷却机制 (Cooldown)**：在前端引入了基于状态的 60 秒倒计时机制。每次点击互动后，按钮会进入冷却状态，按钮文本实时显示倒计时（如 `59s`）。
- **按钮重设计**：加入 `Gamepad2` 图标，并在底部标明 `Free`（免费）。
- **悬停预览 (Tooltip)**：
  - 消耗：`免费 (Free)`
  - 收益：`+5 经验值 (EXP)`
  - 额外说明：`冷却时间: 1分钟`
- **视觉反馈**：点击互动后，宠物头像会触发弹跳动画 (`animate-bounce`)，并飘出闪星 ✨ 图标。

### 3.3 视觉细节提升
- **背景与层次**：为宠物卡片增加了顶部渐变背景色 (`bg-gradient-to-b from-blue-50 to-transparent`)，使其看起来更像一张游戏角色卡。
- **进度条优化**：将经验 (EXP)、能量 (Energy) 和饥饿度 (Hunger) 的进度条改为带有渐变色的更现代的圆角细条 (`bg-gradient-to-r`)。
- **等级标签**：使用带有小图标的专属徽章样式替代了原来的纯文本。

## 4. 技术实现细节
- **图标库**：全面引入 `lucide-react`，使用标准化的 SVG 图标替换了部分 Emoji，提升了界面的专业感。
- **状态管理**：使用 `useState` 和 `useEffect` 管理喂食 (`isFeeding`)、互动 (`isPlaying`) 的临时动画状态和冷却计时器 (`cooldown`)。
- **定时器清理**：严格在 `useEffect` 返回函数中清理 `setTimeout`，防止内存泄漏。
- **容错处理**：增加了 `loading` 和 `error` 状态下的骨架屏 (Skeleton) 设计，提升了弱网环境下的体验。

## 5. 测试结果
- **功能测试**：金币扣除、经验增加、能量/饥饿度更新均与后端状态同步良好。
- **响应式测试**：在移动端 (375px) 和桌面端 (1024px+) 均表现出完美的自适应流式布局。
- **交互测试**：防抖与冷却机制有效拦截了连续恶意点击。
